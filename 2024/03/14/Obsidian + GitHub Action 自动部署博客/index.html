Husky 使用

为了实现我们减少我们编写博客的部署成本，我们可以考虑 GitHub Action 、GitHook等一些实现自动化。

由于我实际静态博客网页代码存在于本地目录中 myblog/.deploy_git中，而编写博客的文件夹位于myblog/source/_posts中，如果在 myblog/.deploy_git 对应的代码仓库使用 GitHub Action 无法完成 hexo 进行 clean generate 等命令。如果以 myblog/ 建立一个远程仓库使用 GitHub Action 不免麻烦。

于是我打算在myblog/ 建立一个本地仓库，并使用 Husky 以便于设置 GitHook 。GitHook 可以在执行代码的 commit、push、rebase 等阶段前触发，做一些前置行为，比如在每次提交代码时候执行一段 shell 脚本，我们正可以在脚本中调用本地 hexo 进行部署。
